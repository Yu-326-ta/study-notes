# UI/UX 設計 — Study Quiz

## 1. デザイン原則

### 1.1 単語帳メタファー

物理的な単語帳と同じ mental model を維持する。

```
┌─────────────────┐
│   表（問題）     │  ← 最初に見える  📎 資料（常時）
└────────┬────────┘
         │ タップ「答えを見る」
         ▼
┌─────────────────┐
│   裏（回答）     │  ← 模範回答 + 要点  📎 資料（常時）
└────────┬────────┘
         │ 自己採点 ○ △ ×
         ▼
      次の問題
```

- **1 画面 1 問** — 情報過多を避け、集中を維持
- **段階的開示** — 模範回答はユーザー操作後に表示。**資料（引用元）は常時 1 タップで開ける**
- **大きなタップ領域** — 移動中でも操作可能

### 1.2 資料への常時アクセス

回答画面に依存せず、**資料はいつでも確認できる** UI を徹底する。

```
┌─────────────────────────────────────┐
│  グローバルナビ                      │
│  [ ホーム ] [ 問題 ] [ 資料 ] [ 統計 ]│  ← 「資料」タブ常設
└─────────────────────────────────────┘

学習セッション（問題フェーズでも表示）:
┌──────────────────────────────────────┐
│  ← 終了    📎 資料    3 / 12         │  ← ヘッダーに常時 📎
├──────────────────────────────────────┤
│  （問題文 — 答えはまだ非表示）        │
└──────────────────────────────────────┘
         │ タップ 📎
         ▼
┌──────────────────────────────────────┐
│  資料（ボトムシート / サイドパネル）   │  ← セッションを離脱せず表示
│  抜粋 + [ Notion で開く ]            │
│  [ 閉じる ]                          │
└──────────────────────────────────────┘
```

| 原則 | 実装 |
| ---- | ---- |
| 資料は学習の補助 | 答えを見る前でも資料 OK（チートではなく再読の導線） |
| 同じ UI で統一 | セッション・一覧・資料タブで同じ `SourcePanel` コンポーネント |
| セッション中断しない | モーダル / ボトムシート。別ページ遷移は資料タブのみ |

### 1.3 記憶定着 UX

| 原則 | UI への落とし込み |
| ---- | ----------------- |
| 考える時間を確保 | 問題表示後、即「答えを見る」を目立たせすぎない |
| フィードバックは自己判断 | 模範回答を見せ、ユーザーが ○△× で採点 |
| 抜けていた点を可視化 | 模範回答を **要点リスト** で自己チェック |
| すぐ再学習できる | × 採点後「もう一度」で即リトライ |
| 達成感 | セッション終了サマリ |
| 本編優先 | 周辺知識は別入口・視覚的にセカンダリ |

### 1.4 問題集の視覚的区別

| 問題集 | ホーム上の扱い | バッジ色（案） |
| ------ | -------------- | -------------- |
| 本編（Notion） | Primary CTA・上部 | 青 `#2563EB` |
| 周辺知識 | 別カード・下部 | グレー `#6B7280` |

### 2.1 S-01 ホーム

**目的:** 今日何をすればよいかを 3 秒で理解させる。

```
┌──────────────────────────────────────┐
│  Study Quiz                    ⚙️   │
├──────────────────────────────────────┤
│  [ ホーム ] [ 問題 ] [ 資料 ] [ 統計 ]│
├──────────────────────────────────────┤
│  🔥 3日連続                          │
│                                      │
│  ── 本編問題集（優先）──             │
│  ┌────────────────────────────────┐  │
│  │  今日の復習  12問              │  │
│  │  [  復習を始める  ]  ← Primary │  │
│  └────────────────────────────────┘  │
│                                      │
│  質問集 42問  プロダクト深掘り 36問   │
│  会社情報 18問  …                    │
│                                      │
│  ── 周辺知識問題集 ──                │
│  ┌────────────────────────────────┐  │
│  │  周辺知識  復習 5問 / 未学習 30 │  │
│  │  [  周辺知識を学ぶ  ]          │  │
│  └────────────────────────────────┘  │
│                                      │
│  📎 [ 資料一覧を見る ]               │
└──────────────────────────────────────┘
```

**UI 要素:**

- Primary CTA は **本編の「今日の復習」1 つだけ**
- グローバルナビ **「資料」** で Notion 資料一覧へ常時アクセス
- 周辺知識は視覚的にセカンダリ（色・位置で優先度差）

### 2.2 S-03 学習セッション（最重要）

#### ステップ 1: 問題

```
┌──────────────────────────────────────┐
│  ← 終了    📎 資料    3 / 12        │
├──────────────────────────────────────┤
│  [本編] [質問集]  Go                  │
│                                      │
│  Go の GC はどの方式で、              │
│  どんな特徴がありますか？             │
│                                      │
│         💭 考えてから見よう           │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  メモ（任意）                  │  │
│  └────────────────────────────────┘  │
│                                      │
│  ┌────────────────────────────────┐  │
│  │      答えを見る                │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

- ヘッダー **📎 資料** は問題フェーズ・回答フェーズ **両方** に常時表示
- タップでボトムシート / サイドパネル（セッション維持）

#### ステップ 2: 模範回答 + 自己採点

```
┌──────────────────────────────────────┐
│  ← 終了          3 / 12              │
├──────────────────────────────────────┤
│  模範回答                            │
│  ─────────────────                   │
│  Go の GC は Concurrent Mark and     │
│  Sweep 方式。停止時間が短い。        │
│                                      │
│  要点チェック                        │
│  □ Concurrent Mark and Sweep        │
│  □ 停止時間（STW）が短い             │
│  □ マイクロサービス向き              │
│                                      │
│  覚えていた程度は？                  │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │ 😊     │ │ 🤔     │ │ 😵     │   │
│  │覚えて  │ │うろ覚え│ │忘れて  │   │
│  │  た    │ │        │ │  いた  │   │
│  └────────┘ └────────┘ └────────┘   │
│                                      │
│  [ 次の問題 → ]                      │
└──────────────────────────────────────┘
```

- 回答画面にも 📎 資料ボタン（ヘッダー共通）
- 「引用元を見る」折りたたみは **廃止** — 資料はヘッダー 📎 に統一

**自己採点ボタン:**

| 評価 | 色 | SRS への影響 |
| ---- | -- | ------------ |
| 覚えていた | 緑 | 復習間隔を延長（1→3→7→14 日…） |
| うろ覚え | 黄 | 短い間隔で再出題（1→2 日） |
| 忘れていた | 赤（控えめ） | 翌日再出題、優先度 UP |

#### ステップ 3: セッション完了

```
┌──────────────────────────────────────┐
│           🎉 セッション完了           │
│                                      │
│     12問中                             │
│     😊 7  🤔 3  😵 2                  │
│                                      │
│     次回復習予定: 明日 5問           │
│                                      │
│  [ ホームに戻る ]  [ 続けて学習 ]    │
└──────────────────────────────────────┘
```

### 2.3 S-02 問題一覧

- **問題集タブ**: 本編 | 周辺知識
- 検索（質問文・タグ）
- フィルタ: カテゴリ / 状態 / タグ
- 各行: 問題タイトル、状態、📎 資料（回答前でもタップ可）
- タップ → 学習開始 / 資料のみ閲覧

### 2.4 S-04 資料一覧・詳細

**目的:** 学習セッション外でも、Notion 資料をいつでも読める。

```
┌──────────────────────────────────────┐
│  資料                          🔍   │
├──────────────────────────────────────┤
│  [ すべて ] [ 質問集 ] [ 深掘り ] …  │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 📄 GCはなぜあるの？            │  │
│  │ 関連問題: 3問                  │  │
│  └────────────────────────────────┘  │
│  ┌────────────────────────────────┐  │
│  │ 📄 使用言語の仕様Go            │  │
│  │ 関連問題: 8問                  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

**詳細画面:**

- 抜粋全文（スクロール）
- [ Notion で開く ↗ ]
- 「この資料の問題を解く」→ 該当問題のみでセッション開始
- 周辺知識問題は `referenceUrl`（公式ドキュメント等）も表示

### 2.5 S-05 統計

- 週間ヒートマップ
- **本編 / 周辺知識** 別の正答率
- 「よく忘れるタグ TOP 5」（本編・周辺それぞれ）

## 3. インタラクション

### 3.1 ジェスチャ（モバイル）

| 操作 | 動作 |
| ---- | ---- |
| 左スワイプ | 次の問題（採点後のみ） |
| 右スワイプ | 前の問題（任意、Phase 2） |
| 長押し | 引用元クイック表示 |

### 3.2 キーボード（デスクトップ）

| キー | 動作 |
| ---- | ---- |
| `Space` | 答えを見る / 次へ |
| `1` / `2` / `3` | 覚えていた / うろ覚え / 忘れていた |
| `S` | 資料パネル開閉（セッション内） |
| `Esc` | セッション終了確認 |

### 3.3 アニメーション

- 問題 ↔ 回答: カードフリップ（300ms、`prefers-reduced-motion` 尊重）
- ページ遷移: 最小限（セッション内は state 更新のみ）
- 自己採点後: 軽い haptic（対応デバイスのみ）

## 4. ビジュアルデザイン

### 4.1 トーン

- **落ち着いた学習ツール** — ゲーミフィケーション過多を避ける
- ダークモード対応（OS 設定連動）
- Notion 風の読みやすさ（余白多め、行間 1.6 以上）

### 4.2 カラーパレット（案）

| 用途 | Light | Dark |
| ---- | ----- | ---- |
| 背景 | `#FAFAF9` | `#191919` |
| カード | `#FFFFFF` | `#252525` |
| Primary | `#2563EB` | `#3B82F6` |
| 覚えていた | `#16A34A` | `#22C55E` |
| うろ覚え | `#CA8A04` | `#EAB308` |
| 忘れていた | `#DC2626` | `#EF4444` |
| 引用元 | `#6366F1` | `#818CF8` |

### 4.3 タイポグラフィ

- 問題文: 18–20px、semibold
- 模範回答: 16px、regular
- 引用元抜粋: 14px、italic、左ボーダー

### 4.4 コンポーネント

| コンポーネント | 用途 |
| -------------- | ---- |
| `QuestionCard` | 問題・回答のフリップカード |
| `SelfGradeButtons` | 3 段階採点 |
| `SourcePanel` | 資料表示（セッション・一覧・資料タブ共通） |
| `SourceList` | 資料一覧 |
| `QuestionSetTabs` | 本編 / 周辺知識切替 |
| `GlobalNav` | ホーム・問題・資料・統計 |
| `ProgressBar` | セッション進捗 |
| `CategoryBadge` | カテゴリ・タグ |
| `ModeCard` | ホームのモード選択 |

## 5. エラー・空状態

| 状態 | 表示 |
| ---- | ---- |
| 復習 0 問 | 「今日の復習は完了です 🎉」+ 未学習モードを提案 |
| 未学習 0 問 | 「すべて学習済みです」+ 苦手モードを提案 |
| localStorage 不可 | 警告バナー「進捗は保存されません」 |
| 問題 0 件 | 「問題データがありません」（開発時） |

## 6. レスポンシブ

| ブレークポイント | レイアウト |
| ---------------- | ---------- |
| `< 640px` | フル幅カード、ボトム固定 CTA |
| `≥ 640px` | 最大幅 `480px` 中央寄せ（単語帳幅） |
| `≥ 1024px` | 左: セッション、右: **資料サイドパネル常時表示可**（折りたたみ可） |
