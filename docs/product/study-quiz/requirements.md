# 用件定義 — Study Quiz

## 1. 概要

### 1.1 プロダクト名（仮）

**Study Quiz**（勉強問題集）

### 1.2 目的

Notion に蓄積した **質問集**（面接 Q&A）と **プロダクト深掘り**（技術選定・設計の深掘り）の内容を、**能動的想起（Active Recall）** によって定着させる。

単語帳のように「問題 → 回答 → 正誤確認」のサイクルを繰り返し、面接・技術深掘りの場面で即座に説明できる状態を目指す。引用元資料は **回答画面に限らず、いつでも参照できる** ようにする。

### 1.3 背景・課題

| 課題 | 本プロダクトでの解決方針 |
| ---- | ------------------------ |
| Notion を読むだけでは定着しない | 問題形式に変換し、自分で答えてから正解を見る |
| どこに書いてあったか思い出せない | 各問題に **引用元（Notion ページ・セクション）** を紐づけ、**資料一覧からいつでも閲覧** 可能にする |
| 本番で聞かれる内容と周辺知識の優先度が違う | **本編問題集**（Notion 由来）と **周辺知識問題集** を分離し、出題・復習の優先度を独立管理 |
| 復習すべき問題が分からない | 自己採点結果をもとに **苦手問題を優先出題** |
| モバイルでもサッと復習したい | 1 問 1 画面・大きなタップ領域・オフライン不要の軽量 UI |

### 1.4 問題集の構成（2 系統）

優先度が異なるため、**2 つの問題集** として独立管理する。

| 問題集 | ID | 優先度 | 説明 |
| ------ | -- | ------ | ---- |
| **本編問題集** | `notion` | **高** | Notion「勉強」配下に記載のある内容を **すべて** 問題化 |
| **周辺知識問題集** | `related` | **低** | Notion で使用している技術に関連するが、Notion 本文にまとまっていない周辺知識 |

#### 本編問題集（`notion`）— 対象コンテンツ

Notion に記載のある関連コンテンツは **漏れなく** 問題化する。

| カテゴリ | 内容例 | 問題化の想定 |
| -------- | ------ | ------------ |
| **質問集** | 面接 Q&A（Go、PR レビュー、キャッチアップ方法など） | 質問 → 模範回答の想起式 |
| **プロダクト深掘り** | GC、SOLID、Spanner、モジュラモノリスなど全トピック | 概念説明・比較・理由説明 |
| **会社情報** | 各社の求人・志望動機・事業理解 | 会社別 Q&A |
| **leetcode** | 計画表・解法メモ | アルゴリズム・解法の想起 |
| **職務経歴書** | 経歴・実績の説明 | エピソード想起 |

> ホームの Primary CTA（今日の復習）・デフォルト出題は **本編問題集のみ** を対象とする。

#### 周辺知識問題集（`related`）— 作成方針

Notion 本編で触れている技術タグを起点に、**本文に書いていないが理解に必要な周辺知識** を問題化する。

| 例（Notion 本編） | 周辺知識問題の例 |
| ----------------- | ---------------- |
| Spanner を採用 | CAP 定理、TrueTime、外部一貫性 |
| Go の GC | 他言語 GC との比較、Mark-Sweep の一般論 |
| Pub/Sub | 少なくとも一度配信、Ordering key |
| Valkey | Redis との関係、キャッシュ戦略の一般論 |

- 本編問題との **タグで関連付け**（例: `relatedTo: ["Spanner"]`）し、必要なら資料から辿れる
- 本編の復習キューには **混ぜない**（明示的に周辺知識モードを選んだときのみ出題）

## 2. ユーザー

### 2.1 想定ユーザー

- 本人（就職・転職活動中のエンジニア）
- 1 人利用を前提（認証・マルチユーザー不要）

### 2.2 利用シーン

| シーン | 期待する使い方 |
| ------ | -------------- |
| 通勤・隙間時間 | 5 分で 10 問、スワイプで次へ |
| 本番前の総復習 | カテゴリ・タグで絞り込み |
| 苦手克服 | 「要復習」フィルタで重点学習 |
| 元ネタの確認 | 学習中・一覧・資料タブから **いつでも** 引用元を開き Notion と照合 |
| 周辺知識の補強 | 本編を優先した後、別モードで周辺知識問題集を学習 |

## 3. 機能要件

### 3.1 必須機能（MVP）

#### F-01 問題一覧・カテゴリ選択

- **問題集**（本編 / 周辺知識）で切り替え — デフォルトは本編
- カテゴリ（質問集 / プロダクト深掘り / 会社情報 等）でフィルタ
- タグ・トピック（例: `Go`, `GC`, `Spanner`）で絞り込み
- 各問題の学習状態（未学習 / 復習待ち / 定着）を一覧表示
- 各行から **資料（引用元）へ直接遷移** 可能（回答前でも可）

#### F-02 学習セッション（コア体験）

1. **問題表示** — 質問文のみ表示（答えは非表示）
2. **回答フェーズ** — ユーザーが心の中またはテキストエリアで回答を考える
3. **答え表示** — 「答えを見る」で模範回答・要点を表示
4. **自己採点** — 「覚えていた / うろ覚え / 忘れていた」の 3 段階（または ○△×）
5. **次の問題へ** — 1 タップで遷移

> 引用元は学習セッション内でも **問題表示中・回答表示中を問わず常時アクセス** 可能（F-03 参照）。

#### F-03 資料・引用元（ソース）閲覧 — **常時アクセス**

引用元資料は **回答画面専用ではない**。以下のすべてから同じ資料に到達できる。

| アクセス経路 | 説明 |
| ------------ | ---- |
| **グローバルナビ「資料」** | Notion ページ単位の資料一覧（S-04） |
| **学習セッション** | 問題・回答のどちらのフェーズでも 📎 ボタンで表示 |
| **問題一覧** | 各行の資料アイコンから直接 |
| **ホーム** | カテゴリ別「資料を読む」ショートカット |

各 **資料（SourceDocument）** に保持する項目:

| 項目 | 説明 |
| ---- | ---- |
| `id` | 資料の一意 ID（複数問題から参照可） |
| `title` | 元ページ名（例: 「GCはなぜあるの？」） |
| `url` | Notion 公開 URL（新規タブ） |
| `excerpt` | アプリ内表示用抜粋（全文 or 長文） |
| `section` | ページ内セクション（任意） |
| `questionSet` | `notion` 資料 or `related` 用の参考リンク（周辺知識は外部 URL 可） |

各 **問題** は `sourceId` で資料に紐づく。1 資料 → 複数問題の関係を許容する。

> 資料は **信頼の根拠** かつ **再読の導線**。学習フローを中断せず、いつでも確認できることを最優先する。

#### F-04 進捗・復習管理（localStorage）

DB は使わず、ブラウザ `localStorage` に保存する。

| 保存項目 | 用途 |
| -------- | ---- |
| 最終回答日時 | 間隔反復の計算 |
| 自己採点結果（履歴） | 正答率・苦手分析 |
| 次回復習予定日 | 復習キュー |
| セッション途中状態 | 中断・再開 |

#### F-05 出題モード

**本編問題集（優先）**

| モード | 説明 |
| ------ | ---- |
| **今日の復習** | 本編の復習予定日 ≦ 今日（**デフォルト Primary**） |
| **未学習** | 本編で一度も回答していない問題 |
| **苦手** | 本編で直近「忘れていた」が多い問題 |
| **すべて** | 本編全問 |
| **カテゴリ指定** | 質問集 / プロダクト深掘り / 会社情報 等 |

**周辺知識問題集（別枠）**

| モード | 説明 |
| ------ | ---- |
| **周辺知識の復習** | 周辺知識セットの SRS キュー |
| **周辺知識・未学習** | 周辺知識セットの未学習 |
| **タグ連動** | 本編で苦手なタグに紐づく周辺知識を提案（任意） |

> 本編と周辺知識は **同一セッションに混在させない**（優先度を保つため）。例外: ユーザーが明示的に「混合モード」を ON にした場合のみ（Phase 2）。

#### F-06 統計ダッシュボード（簡易）

- 総問題数 / 学習済み数 / 定着率
- カテゴリ別の正答率（自己採点ベース）
- 連続学習日数（ストリーク）

### 3.2 問題形式

Phase 1 でサポートする形式。

| 形式 | 説明 | 例 |
| ---- | ---- | -- |
| **自由想起**（主） | 質問を見て、自分の言葉で答えを考え、模範回答と比較 | 「Go の GC はどんな方式？」 |
| **要点チェック** | 模範回答を複数の要点に分解し、自己チェックリストとして表示 | □ Concurrent Mark and Sweep □ 停止時間が短い |
| **比較問題** | 2 つの概念の違いを説明 | 「Goroutine と OS スレッドの違いは？」 |

Phase 2 以降（任意）: 4 択、穴埋め、並べ替え。

### 3.3 除外機能（スコープ外）

- ユーザー認証・ログイン
- サーバー DB・API
- 問題の UI 上での編集（問題データは Git 管理の JSON/Markdown）
- Notion とのリアルタイム同期
- 他ユーザーとの共有・ランキング

## 4. 非機能要件

### 4.1 技術

| 項目 | 要件 |
| ---- | ---- |
| フレームワーク | Next.js（App Router）+ TypeScript |
| データ | 静的 JSON（`src/data/questions/` 等） |
| 進捗 | `localStorage`（キーは `study-quiz-progress` 等） |
| スタイル | Tailwind CSS、モバイルファースト |
| デプロイ | Vercel 等の静的ホスティング |

### 4.2 パフォーマンス

- 初回表示 2 秒以内（問題データはビルド時にバンドル or 動的 import）
- 問題切り替え 200ms 以内（ページ遷移なし、同一画面内更新）

### 4.3 アクセシビリティ

- キーボード操作: `Space`（答え表示）、`1/2/3`（自己採点）、`Enter`（次へ）
- 十分なコントラスト比、フォーカス表示
- タップ領域 44px 以上（モバイル）

### 4.4 データ永続性

- `localStorage` クリアで進捗消失 — ユーザー向けに「エクスポート/インポート（JSON）」を Phase 2 で検討
- 問題データ更新時、ID 不変であれば進捗は維持

## 5. 学習設計方針（記憶定着）

科学的に効果が高いとされる手法を UI に組み込む。

| 手法 | アプリでの実装 |
| ---- | -------------- |
| **能動的想起** | 答えを見る前に必ず「考える時間」を設ける（空の回答エリア or タイマー） |
| **自己採点** | 正誤をシステムが判定せず、ユーザーが ○△× で評価（言い回しの揺れを許容） |
| **間隔反復（SRS）** | ○→ 間隔延長、×→ 翌日再出題 など簡易 SM-2 風スケジュール |
| **インターリービング** | 本編内ではカテゴリ混在ランダム（同トピック連続を避ける）。周辺知識は別キュー |
| **メタ認知** | 自己採点後に「どこが抜けていたか」を任意メモ（localStorage） |

詳細 UI は [ui-ux-spec.md](./ui-ux-spec.md) を参照。

## 7. MVP スコープ

### Phase 1（MVP）

- [ ] **本編問題集**: Notion 全カテゴリから問題化（質問集・プロダクト深掘りを先行、続けて会社情報等）
- [ ] **周辺知識問題集**: 本編タグに紐づく周辺知識を別 JSON で管理（初期 20 問程度）
- [ ] 学習セッション（F-02）+ **資料常時アクセス**（F-03）
- [ ] 資料一覧画面（S-04）
- [ ] localStorage 進捗 — **問題集別** に独立保存（F-04）
- [ ] 出題モード: 本編（今日の復習 / 未学習 / すべて）+ 周辺知識（別入口）
- [ ] モバイル対応 UI

### Phase 2

- [ ] 統計ダッシュボード拡充（問題集別）
- [ ] 進捗 JSON エクスポート/インポート
- [ ] 本編の苦手タグ → 周辺知識問題の提案
- [ ] 4 択・穴埋め形式
- [ ] Notion からの半自動問題生成スクリプト

### Phase 3

- [ ] PWA（オフライン学習）
- [ ] 混合モード（任意）

## 9. 用語集

| 用語 | 定義 |
| ---- | ---- |
| 問題（Question） | ユーザーに提示する質問と、それに対する模範回答のセット |
| 本編問題集 | Notion 記載内容から作成した問題（優先度高） |
| 周辺知識問題集 | 本編技術に関連するが Notion 外の知識（優先度低・別管理） |
| 資料（SourceDocument） | Notion ページ等の引用元。問題に独立し、いつでも閲覧可能 |
| 引用元（Source） | 問題が参照する資料（`sourceId`） |
| 自己採点 | ユーザーが模範回答を見た後に行う ○△× 評価 |
| 復習予定日 | SRS に基づき次に出題する日 |
| セッション | 連続して複数問を解く一連の学習単位 |
